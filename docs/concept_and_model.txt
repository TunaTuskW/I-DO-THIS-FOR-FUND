MACRO ANALYST AGENT - CONCEPT AND MODEL
=======================================

1. WHAT IT IS
This is a 24/7 autonomous quantitative agent that monitors global markets, processes data through deep learning classifiers, and generates institutional-grade macro reports. It runs completely automatically in the background using your computer's scheduler. The presentation layer uses an ultra-minimal "Brutalist Terminal" text-block layout to ensure a data-dense, cross-platform reading experience designed for instant mobile parsing.

2. THE ENTERPRISE DECOUPLED ARCHITECTURE (v4.5.0 Upgrade)
Following the v4.5.0 Enterprise upgrade, the agent is refactored from a monolithic script into a clean, decoupled pipeline that integrates abstract interfaces, specialized adapters, a partitioned data lake, and isolated math engines:

- Core Interfaces: Strict contracts (`DataBroker` and `LLMProvider` under `src/interfaces/`) enforce structural modularity and prevent vendor lock-in.
- Modular Adapters: Implementing providers (`YahooAdapter` and `GeminiAdapter` under `src/adapters/`) ingest price series, bond yields, and semantically score news headlines.
- Partitioned Data Lake: The database manager (`LakeManager` under `src/data_lake/`) writes tabular Parquet tables and unstructured JSONL snapshots to daily partitioned directories (`data/raw/YYYY/MM/DD/`).
- Decoupled engines: Logic is isolated into standalone calculators under `src/engines/` (`feature_engine.py` for indicators, `hmm_engine.py` for HMM regime scoring, and `risk_engine.py` for Kalman/Entropy/Kelly sizing).
- Contextual Observability: A structured console and JSONL logging framework (`src/observability/logger.py`) maps operations across distinct components, piping events to `data/logs/system_events.jsonl`.
- Consensus Conductor: The central orchestrator (`src/fetch_market_data.py`) leverages dependency injection to initialize adapters, engines, and managers, executing the sequence smoothly.

3. THE END-TO-END WORKFLOW
- Context Logging: Conductor initializes, spinning up custom component-level loggers to write system events.
- Ingestion & Parquet Partitioning: YahooAdapter fetches historical price/yield series. Conductor invokes LakeManager to save daily and hourly series directly to the Data Lake as partitioned Parquet tables (`raw_daily_ohlcv.parquet` and `raw_hourly_ohlcv.parquet`).
- Feature Construction: FeatureEngine computes stats, GARCH conditional volatility, Gold-to-Silver momentum, Institutional Crypto MFI, and MCS stress metrics.
- Mathematical Scoring: Conductor pipes feature vectors to HMMEngine and RiskEngine to determine regime probabilities, update Kalman filter matrices, measure Shannon entropy, and calculate exposure targets using Fractional Kelly.
- NLP News Scoring: GeminiAdapter fetches RSS headlines and semantically decodes geopolitical shock and liquidity drain probabilities via JSON-constrained LLM parsing.
- Snapshot Lake Log: Conductor packages the complete market state into `data/market_snapshot.json` and writes it as an unstructured daily-partitioned log (`market_snapshot.jsonl`) to the Data Lake.
- Presentation & Delivery: Report compilers parse the snapshot into Brutalist markdown and push the briefing to Discord/Telegram.
- Cryptographic Proof: Snapshots are signed and appended to `logs/immutable_chain.log` (TruChain) to guarantee historical audit integrity.

4. THE MATH & MODELS BEHIND THE SCENES
- Macro Condition Score (MCS): A composite number measuring overall market stress based on stock momentum, interest rates, energy prices, and cross-asset behavior (FeatureEngine).
- Hidden Markov Models (HMM) & Kalman Filters: These algorithms smooth out daily noise to figure out the "true" underlying regime (HMMEngine & RiskEngine).
- Deep Classifier (MLP): A trained neural network that double-checks the HMM. 
- Institutional Footprinting: Natively supports advanced institutional footprinting, including a 20-day continuous Institutional Heat Index (IHI).
- Semantic NLP NewsSignalVector (Google Gemini LLM Engine): 
  Operates autonomously via a structured, JSON-constrained Gemini API pipeline to parse live news headlines for strict quantitative shock boundaries.
  * Parameter Decoding: Gemini parses headlines and outputs strict JSON limits: `liquidity_drain_probability` and `geopolitical_shock_magnitude` (floats in [0.0, 1.0]).
  * Shock Assignment: Discourages naive directional bias, translating geopolitical shock magnitude into compound volatility shocks (VIX impact) and liquidity drain probabilities into conditional noise multipliers.
  * Geopolitical Event Flag: Event flag pings are automatically set to 1 if the geopolitical shock magnitude sweeps past the critical **0.70 boundary**, scaling active consensus requirements to 0.65.
- Hyperparameter Tuning Meta-Agent (tune_hyperparameters.py):
  * Dynamic Parameter Tuning: A standalone meta-agent scheduled to run weekly. It feeds massive macroeconomic text blocks (such as FOMC meeting minutes or Beige Books) to the Gemini LLM.
  * Configuration Tuning: Translates central bank transcripts into concrete mathematical half-life decay thresholds and calibration scalars, dynamically overwriting active constants in `fetch_market_data.py` via the local `tuning_configs.json` configuration file.

5. ADVANCED MODEL GOVERNANCE & TACTICAL ASSUMPTIONS
- Kelly Sizing Decay Penalty: The algorithm targets exposure using the Fractional Kelly Criterion, heavily penalized by historical Brier Score calibration. Furthermore, if a regime outlives its mathematical "half-life" (e.g., 11 days for RISK_ON), an exponential decay penalty forcefully scales the portfolio exposure down, enforcing systematic profit-taking before transitions occur.
- 1.5 Std Dev Dynamic Boundaries: Continuously tracks volatility thresholds to automatically invalidate outdated narratives upon statistical sweeps.

6. ESCALATION SYSTEM
The agent respects your time and won't spam you with panicky alerts. It only flags urgency when the math demands it:
- ROUTINE: Normal market updates.
- ELEVATED: Moderate volatility spikes, sudden currency squeezes, or early-warning transition signals.
- CRITICAL: Extreme mathematical stress (e.g., VIX > 30, simultaneous stock and bond crashes). Discord will ping you heavily for these.

7. THE AUTOMATED SCHEDULE
The agent manages three distinct workflows natively via bash scripts (run_4h.sh, run_daily.sh, run_weekly.sh) executed by cron:
- The 4-Hour Briefing: A quick pulse-check of the active market session.
- The Daily Roll: At midnight, it pushes a master document containing the last 72 hours of updates.
- The Sunday Synthesis: A deep-dive research report summarizing the entire week and previewing the week ahead.

8. OFFLINE MODEL TRAINING & BACKTESTING
The deep learning models (HMM and MLP) are not static. The agent includes a robust, double-engine offline training pipeline (train_models.py) and historical backtesting engine (backtest.py) to periodically adapt to evolving market regimes:

- The HMM Regime Training Pipeline:
  * Ingests 5 years of daily historical data across Equities (SPX), Volatility (GARCH volatility proxy), FX/Treasuries (DXY, USDCAD, 10Y yields, 10Y-2Y yield spread), Commodities (WTI Crude, Gold, Silver), and a Liquidity/Crypto Flow proxy.
  * Standardizes the aligned 10-dimensional feature vector using a StandardScaler.
  * Fits a Gaussian Hidden Markov Model (GaussianHMM) configured with N_HIDDEN_STATES = 6, a full covariance matrix structure, and 500 maximum EM iterations.
  * Dynamically maps states by checking their emission means across daily SPX returns, 10-year yield shifts, and WTI moves:
    - RISK_ON_EXPANSION: Pos. SPX return (>0.3%) & rising yields (>0.01%).
    - LIQUIDITY_DRIVEN_RALLY: Pos. SPX return (>0.3%) & falling yields (<-0.01%).
    - STAGFLATION_STRESS: Neg. SPX return (<-0.2%) & rising energy price (>0.5%).
    - RATE_SHOCK: Neg. SPX return (<-0.2%) & expanding bond yields (>0.02%).
    - DEFLATION_FEAR: Neg. SPX return (<-0.2%) & falling energy price (<-0.3%).
    - NEUTRAL_TRANSITIONAL: Stable market conditions.

- The Supervised MLP Deep Classifier Pipeline:
  * Targets a 5-day forward cumulative SPX return: Risk-On (y = 1, return > 1.5%), Risk-Off (y = 0, return < -1.5%), and Transitional (y = 2, otherwise).
  * Train package runs a multi-layer perceptron neural network (MLPClassifier) with a (16, 8) hidden layer topology.
  * Leverages a ReLU activation function, Adam optimization solver, and early stopping to protect against overfitting.

- The Historical Backtest Engine (backtest.py):
  * Loads the active models and decodes 2 years of historical daily market features into their mapped state label sequences.
  * Measures the daily mean returns and annualizes SPX/WTI metrics across all 6 regimes, compiling a comprehensive, tabular performance audit (backtest_results.md) detailing frequencies, annualized returns, and yield moves (in basis points per day) to verify mathematical edge before live deployment.
- The Interactive Math Visualizers (visualize_math_4h.ipynb & visualize_math_1w.ipynb):
  * Displays dynamic visual analysis overlaid natively in VS Code, allowing visual audit of the quantitative engines for 4-Hour Briefings and Weekly Synthesis cycles.
  * Overlays HMM-inferred market regimes directly onto the S&P 500 price index.
  * Plots heatmaps of cross-asset structural stress vectors, including the **Volatility Term Structure** (VIX9D vs VIX curves) to expose backwardation points.
  * **Gemini Geopolitical Shock Plots (Section 6):** Dynamically plots semantic news shocks against a horizontal red line indicating the critical **0.70 Geopolitical Shock Trigger** boundary.
  * Visualizes the Epistemic Kelly sizing allocation curves and active exponential half-life duration decay factors.
  * Embeds dynamic live report rendering (Section 5) directly inside VS Code below the mathematical charts.

9. DATA PRIVACY & ZERO-SHARING SECURITY PROTOCOLS
To protect proprietary quantitative intelligence and restrict leakage of sensitive credentials, the system enforces a local-only encapsulation boundary:
- Strict Ignored Enclaves: The `.gitignore` config explicitly encapsulates all private credentials (`api_keys.json`), tuned configurations (`tuning_configs.json`), private asset snapshots (`market_snapshot.json`), forecast accuracies (`predictions_history.json`), model weights (`models/hmm_model.pkl`, `models/mlp_model.pkl`), run-time logs (`logs/`), and generated briefings (`reports/updates/`).
- Decoupled Example Schemas: Default templates (e.g., `api_keys.example.json`, `market_snapshot.example.json`) are committed to establish skeleton frameworks, which are duplicated locally to active formats. This ensures developers can execute backtests, Gemini news analysis, and offline hyperparameter tuning entirely offline and locally with zero risk of committing operational intelligence or API credentials to public repositories.

By anchoring the system to a rigid mathematical training and verification backbone, the pipeline provides institutional-quality insights with deterministic reliability, entirely free from LLM hallucinations.
